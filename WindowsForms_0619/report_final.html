1. 과제 개요

본 과제는 [실습6] 리프트 제어를 바탕으로, 컨베이어 위의 물체를 두 개의 실린더와 두 개의 리프트로 순환시키는 제어 프로그램을 구현하는 것입니다. 프로그램은 크게 두 가지 동작 모드를 가집니다. 첫째는 버튼 한 번으로 전체 공정을 자동으로 진행하는 자동 운전이고, 둘째는 각 실린더와 리프트를 사용자가 직접 하나씩 제어하는 수동 조작입니다.

구현은 C# 윈도우 폼(Windows Forms) 응용 프로그램으로 작성하였으며, PLC 시뮬레이터와 통신하여 센서 값을 읽고 출력 신호를 내보내는 방식으로 동작합니다. 본 보고서에서는 시스템 구성, 컴포넌트별 핵심 코드, 그리고 시나리오별 동작 검증 결과를 정리하였습니다.

과제에서 요구한 사항은 다음과 같으며, 각 항목을 본문에서 구현 내용과 함께 다루었습니다.

  • 자동 운전 시 전체 시퀀스가 자동으로 진행될 것
  • 수동 조작이 각 컴포넌트(실린더·리프트)별로 가능할 것
  • GUI에 현재의 상태를 모두 표시할 것
  • 보고서를 시나리오별 동작에 관하여 검증할 것
  • 보고서에 컴포넌트별 핵심 코드를 첨부하여 설명할 것

2.2 장치 입출력 정의

PLC에서 X 디바이스는 센서 등 입력 신호를, Y 디바이스는 실린더·리프트로 나가는 출력 신호를 의미합니다. 각 신호는 16비트 워드 안의 한 비트(bit)에 대응하며, 본 프로그램은 입력 워드 X0와 출력 워드 Y0를 한 번에 읽고 써서 비트별로 처리합니다. 구현에 사용한 입출력 대응은 다음과 같습니다.

컴포넌트출력(Y) — 동작 지시입력(X) — 완료/감지 센서
B실린더 전진 / 후진Y1 / Y2X02 / X03
C실린더 전진 / 후진Y3 / Y4X04 / X05
리프트A 상승 / 하강Y5 / Y6X06 / X07
리프트B 상승 / 하강Y7 / Y8X08 / X09
리프트센서 A / B (물체 감지)-XA / XB
과제에서 제공된 [동작정보] 문서에는 C실린더 완료 센서가 전진 X05·후진 X04, 리프트B 출력이 상승 Y8·하강 Y7로 표기되어 있습니다. 다만 실제 시뮬레이터에 연결하여 각 출력을 인가하고 센서 응답을 확인한 결과, 위 표와 같이 동작하는 것을 확인하였습니다. 이에 자동 시퀀스가 정상적으로 순환하도록 실제 동작에 맞추어 매핑하였습니다.

3. 자동 운전 구현

자동 운전은 “실린더로 물체를 리프트까지 밀고, 리프트의 물체 감지 센서가 켜지면 실린더를 후진시킨 뒤 리프트를 올리거나 내린다”는 동작을 반복합니다. 전체 순환 경로는 다음과 같습니다.

B실린더 전진 → 리프트B 센서 감지 → 리프트B 하강 → C실린더 전진 → 리프트A 센서 감지 → 리프트A 상승 → (반복)

이 과정을 일정한 순서대로 진행하기 위해 상태 머신(state machine) 방식을 사용하였습니다. 상태 머신이란 현재 단계를 하나의 값으로 기억해 두고, 센서 조건이 충족되면 다음 단계로 넘어가는 구조를 말합니다. 각 단계는 AutoStep 열거형으로 정의하였습니다.

3.1 핵심 코드

코드설명역할
enum AutoStep { HomeB, HomeC, ReadyLiftBUp, PushToLiftB, RetractBAfterDetect, MoveLiftBDown, ReadyLiftADown, PushToLiftA, RetractCAfterDetect, MoveLiftAUp } 자동 운전의 각 단계를 이름으로 정의한 열거형입니다. 순환 시퀀스를 단계별로 구분해 흐름을 명확히 합니다.
case AutoStep.PushToLiftB: if (liftBOn) autoStep = AutoStep.RetractBAfterDetect; else { WriteY(Y_B_FORWARD); label1.Text = "자동: B실린더 전진..."; } break; 리프트B 센서가 켜질 때까지 B실린더를 전진시키고, 감지되면 후진 단계로 넘어갑니다. 센서 조건에 따라 출력을 내보내고 다음 단계로 전환합니다.
case AutoStep.MoveLiftBDown: if (liftBDown) autoStep = AutoStep.ReadyLiftADown; else WriteY(Y_LIFTB_DOWN); break; 리프트B 하강 출력을 내보내고, 하강 완료 센서가 켜지면 다음 단계로 이동합니다. 완료 센서로 동작 종료를 판단합니다.
case AutoStep.MoveLiftAUp: if (liftAUp) autoStep = AutoStep.ReadyLiftBUp; else WriteY(Y_LIFTA_UP); break; 리프트A 상승이 끝나면 다시 첫 단계(ReadyLiftBUp)로 돌아갑니다. 시퀀스를 반복(순환)시킵니다.
private void button10_Click(...) { autoMode = true; autoStep = AutoStep.HomeB; SetManualButtons(false); label1.Text = "모드: 자동운전 중"; } 자동시작 버튼을 누르면 자동 모드를 켜고 첫 단계부터 시작합니다. 자동 운전 진입 및 수동 버튼 잠금을 담당합니다.

※ HomeC 단계에서는 시작 시점의 센서 상태를 확인하여, 이미 물체가 특정 위치에 있으면 해당 단계로 건너뛰도록 처리해 어느 상태에서 시작해도 시퀀스가 이어지도록 하였습니다.

4. 수동 조작 구현

수동 조작은 8개의 버튼으로 각 실린더와 리프트를 개별 제어합니다. 한 컴포넌트는 전진/후진(또는 상승/하강)이 동시에 켜지면 안 되므로, 한 비트를 켤 때 반대 비트는 끄도록 처리하였습니다. 이를 SetOutput 함수에 모아 두고, 각 버튼은 자신의 출력 비트만 넘겨주도록 구성하였습니다.

4.1 핵심 코드

코드설명역할
private void SetOutput(int onBit, int offBit) { ushort cur = (ushort)output; cur = (ushort)((cur | onBit) & ~offBit); WriteY((short)cur); } 현재 출력값에서 지정한 비트는 켜고 반대 비트는 끈 뒤 내보냅니다. 한 컴포넌트의 전진/후진 동시 출력을 방지합니다.
private void button1_Click(...) { RunManualCommand(Y_B_FORWARD, Y_B_BACKWARD, label4, "B실린더: 전진 명령"); } B실린더 전진 버튼입니다. 켤 비트와 끌 비트, 상태 라벨을 함께 전달합니다. 버튼별 개별 컴포넌트 제어를 담당합니다.
private void RunManualCommand( short onBit, short offBit, Label statusLabel, string text) { SetOutput(onBit, offBit); statusLabel.Text = text; } 출력을 내보내고 해당 컴포넌트의 라벨에 조작 내용을 표시합니다. 8개 버튼이 공통으로 사용하는 처리부입니다.
private void SetManualButtons( bool enabled) { button1.Enabled = enabled; ... // button2 ~ button8 } 수동 버튼 8개를 한 번에 켜고 끕니다. 연결 전·자동 운전 중에는 수동 조작을 잠급니다.

※ 각 버튼은 같은 방식으로 연결되어 있습니다. B·C실린더는 전진/후진, 리프트A·B는 상승/하강 버튼이 각각 자신의 출력 비트를 사용합니다.

5. 상태 표시(GUI) 구현

화면의 상태 표시는 타이머가 250ms마다 입력 워드 X0를 한 번 읽어, 각 비트를 분리한 뒤 라벨에 글자로 나타내는 방식입니다. 이 갱신은 자동·수동 모드와 관계없이 항상 수행되므로, 어떤 모드에서든 현재 센서와 각 컴포넌트의 상태를 확인할 수 있습니다.

5.1 핵심 코드

코드설명역할
int s; if (!TryReadSensors(out s)) { label1.Text = "PLC 통신 대기/재시도 중"; return; } 센서 워드를 읽고, 실패하면 안내만 표시한 뒤 이번 주기를 건너뜁니다. 읽기 실패 시에도 프로그램이 멈추지 않게 합니다.
bool bFwdDone = (s & 0x0004) != 0; bool liftAOn = (s & 0x0400) != 0; bool liftBOn = (s & 0x0800) != 0; // ... (각 센서 비트 분리) 읽어온 워드에서 각 센서에 해당하는 비트를 꺼냅니다. 16비트 묶음을 개별 신호로 분해합니다.
label4.Text = "B실린더: " + (bFwdDone ? "전진 완료" : bBackDone ? "후진 완료" : "동작 중"); label2.Text = "리프트센서 A: " + (liftAOn ? "ON (감지)" : "OFF"); 각 컴포넌트의 현재 상태를 사람이 읽을 수 있는 문구로 표시합니다. 실린더·리프트·센서 상태를 화면에 출력합니다.

6. 통신 및 예외 처리

시뮬레이터와의 통신은 순간적으로 실패하거나, 타이머 동작이 겹쳐 호출되면 오류가 발생할 수 있습니다. 이를 위해 세 가지 처리를 두었습니다. 첫째, 타이머 처리 중에는 다음 타이머가 끼어들지 못하도록 막았습니다. 둘째, 읽기·쓰기 시 예외가 나도 프로그램이 종료되지 않도록 처리하고 다음 주기에 다시 시도하도록 하였습니다. 셋째, 직전과 같은 출력값은 다시 보내지 않아 불필요한 통신을 줄였습니다.

6.1 핵심 코드

코드설명역할
if (ticking || !plcConnected || closing) return; ticking = true; timer1.Stop(); try { TickBody(); } catch { } finally { ticking = false; if (plcConnected && !closing) timer1.Start(); } 처리 중에는 ticking 값으로 중복 실행을 막고, 예외가 나도 안전하게 다음 주기를 준비합니다. 타이머 재진입과 통신 오류로 인한 중단을 방지합니다.
if (!force && word == lastWrittenOutput) return true; ... WriteDeviceBlock2("Y0",1,ref ...); lastWrittenOutput = word; 직전에 보낸 값과 같으면 보내지 않고, 새 값일 때만 출력합니다. 통신 횟수를 줄여 안정성을 높입니다.
private void Form1_FormClosing(...) { closing = true; autoMode = false; timer1.Stop(); if (plcConnected) WriteY(0, true); try { control.Close(); } catch { } } 프로그램을 닫을 때 출력을 모두 끄고 연결을 정리합니다. 종료 시 남는 출력 신호와 연결을 안전하게 해제합니다.

7. 시나리오별 동작 검증

구현한 프로그램을 PLC 시뮬레이터에 연결하여 아래 시나리오대로 동작을 확인하였습니다. 검증은 화면의 상태 라벨과 시뮬레이터의 동작을 함께 관찰하는 방식으로 진행하였습니다.

No.시나리오조작 / 조건확인된 결과
1PLC 연결[연결] 버튼 클릭“연결되었습니다” 안내 후 모드가 ‘수동(대기)’로 바뀌고 수동 버튼이 활성화됨
2자동 운전 진입[자동시작] 버튼 클릭모드가 ‘자동운전 중’으로 바뀌고 수동 버튼이 모두 비활성화됨
3자동 시퀀스 순환자동 운전 유지 관찰상태 라벨이 ‘리프트B 상승 → B실린더 전진 → 리프트B 하강 → … → C실린더 후진’ 순으로 진행되며 순환함

7상태 표시전 과정 동안 라벨 관찰센서 A·B 및 각 컴포넌트 상태가 실시간으로 갱신됨

7.1 실행 화면

그림 1. 연결 직후 — 모드 ‘수동(대기)’, 수동 버튼 활성화, 각 컴포넌트 상태 표시 (시나리오 1)
그림 2. 자동 운전 중 — 상태 라벨 ‘자동: B실린더 전진’, 수동 버튼 비활성화 (시나리오 2·3)
그림 3. 자동 운전 진행 — 라벨이 ‘자동: 리프트B 하강’ 단계로 넘어가며 순환함 (시나리오 3)
그림 4. 수동 조작 — [B실린더 전진] 클릭 후 ‘B실린더: 전진 완료’, 수동 버튼 활성화 (시나리오 5)

※ 화면 우측 상단 라벨은 자동 모드에서는 현재 진행 단계를, 수동 모드에서는 모드 또는 통신 폴링 상태를 표시합니다. 그림 1·4의 ‘PLC 통신 대기/재시도 중’ 문구는 한 주기의 읽기 재시도 안내로, 센서 라벨은 정상적으로 갱신되고 있습니다.

8. 결론

본 과제에서는 리프트 제어 시스템을 자동 운전과 수동 조작 두 모드로 구현하였습니다. 자동 운전은 상태 머신으로 전체 시퀀스를 순서대로 진행하도록 하였고, 수동 조작은 각 실린더와 리프트를 버튼별로 개별 제어할 수 있도록 하였습니다. 또한 모든 컴포넌트의 상태를 화면에 표시하여 동작을 확인할 수 있게 하였습니다.

구현 과정에서 일부 입출력 번호가 문서와 실제 시뮬레이터 동작 사이에 차이가 있어, 직접 출력을 인가하고 센서 응답을 확인하면서 매핑을 맞추는 작업이 필요하였습니다. 이 과정을 통해 자동 시퀀스가 끊김 없이 순환하는 것을 확인하였고, 수동 조작과 상태 표시도 의도대로 동작함을 검증하였습니다. 다만 통신이 순간적으로 지연될 때 우측 상단 라벨에 재시도 안내가 잠시 남는 점은 추후 보완할 수 있는 부분입니다.

부록 A. 전체 소스 코드 (Form1.cs)

using System;
using System.Windows.Forms;
using ACTMULTILIB_K;

namespace WindowsForms_0619
{
    public partial class Form1 : Form
    {
        // PLC 시뮬레이터랑 통신하는 객체
        ActEasyIF control = new ActEasyIF();

        short output = 0;       // 지금 Y0에 내보내는 값
        bool autoMode = false;  // 자동운전 중이면 true
        bool plcConnected = false;
        bool closing = false;   // 폼 닫는 중

        // 직전에 Y0에 쓴 값. 똑같으면 또 안 보내려고 기억해둠
        short lastWrittenOutput = short.MinValue;

        // Y 출력 비트
        const short Y_B_FORWARD = 0x0002;  // Y1 B전진
        const short Y_B_BACKWARD = 0x0004; // Y2 B후진
        const short Y_C_FORWARD = 0x0008;  // Y3 C전진
        const short Y_C_BACKWARD = 0x0010; // Y4 C후진
        const short Y_LIFTA_UP = 0x0020;   // Y5 리프트A 상승
        const short Y_LIFTA_DOWN = 0x0040; // Y6 리프트A 하강
        const short Y_LIFTB_UP = 0x0080;   // Y7 (시뮬에서 이게 리프트B 상승, 완료는 X8)
        const short Y_LIFTB_DOWN = 0x0100; // Y8 (시뮬에서 이게 리프트B 하강, 완료는 X9)

        // 자동운전 단계. 실린더로 밀어서 센서 감지되면 후진하고 리프트 올리고 내림
        enum AutoStep
        {
            HomeB = 0,
            HomeC = 1,
            ReadyLiftBUp = 2,
            PushToLiftB = 3,
            RetractBAfterDetect = 4,
            MoveLiftBDown = 5,
            ReadyLiftADown = 6,
            PushToLiftA = 7,
            RetractCAfterDetect = 8,
            MoveLiftAUp = 9
        }

        AutoStep autoStep = AutoStep.HomeB;

        // 타이머가 겹쳐 돌면 통신이 꼬여서 가끔 에러남. 한 번에 하나만 돌게 막는 용도
        bool ticking = false;

        public Form1()
        {
            InitializeComponent();
        }

        private void Form1_Load(object sender, EventArgs e)
        {
            control.ActLogicalStationNumber = 1; // 스테이션 번호 (Open 전에 세팅해야 함)

            timer1.Interval = 250;      // 너무 빨리 폴링하면 통신 불안정해서 좀 늘림
            timer1.Enabled = false;
            button10.Enabled = false;   // 자동시작
            button11.Enabled = false;   // 자동정지
            SetManualButtons(false);    // 연결 전엔 버튼 다 잠금

            label1.Text = "모드: 대기";
        }

        // 연결 버튼
        private void button9_Click(object sender, EventArgs e)
        {
            int openResult;
            try
            {
                openResult = control.Open(); // 0이면 연결 성공
            }
            catch
            {
                openResult = -1;
            }

            if (openResult == 0)
            {
                plcConnected = true;
                output = 0;
                lastWrittenOutput = short.MinValue;
                WriteY(0, true);          // 전에 켜져있던 출력 있으면 초기화

                MessageBox.Show("연결되었습니다.");

                timer1.Enabled = true;
                button10.Enabled = true;
                button11.Enabled = true;
                button9.Enabled = false;    // 또 누르지 못하게
                SetManualButtons(true);

                label1.Text = "모드: 수동(대기)";
            }
            else
            {
                MessageBox.Show("연결 실패하였습니다.");
            }
        }

        // 자동시작 버튼
        private void button10_Click(object sender, EventArgs e)
        {
            autoMode = true;
            autoStep = AutoStep.HomeB; // 처음엔 실린더 원점부터 맞추기
            SetManualButtons(false);   // 자동 중엔 수동 못 누르게
            // 여기서 MessageBox 띄우면 타이머가 겹쳐 돌면서 에러나서 라벨만 바꿈
            label1.Text = "모드: 자동운전 중";
        }

        // 자동정지 버튼 (연결은 그대로 두고 자동만 끔)
        private void button11_Click(object sender, EventArgs e)
        {
            autoMode = false;
            WriteY(0);                  // 출력 끄기
            SetManualButtons(true);     // 다시 수동 가능
            label1.Text = "모드: 수동(대기)";
        }

        // 타이머: 일정 간격마다 센서 읽어서 화면 갱신, 자동이면 시퀀스 진행
        private void timer1_Tick(object sender, EventArgs e)
        {
            // 앞 틱이 아직 통신 중이면 이번 건 그냥 넘김
            if (ticking || !plcConnected || closing) return;
            ticking = true;
            timer1.Stop();
            try
            {
                TickBody();
            }
            catch
            {
                // 통신 잠깐 튀어도 프로그램 안 죽게. 다음 틱에서 다시 시도됨
            }
            finally
            {
                ticking = false;
                if (plcConnected && !closing)
                {
                    timer1.Start();
                }
            }
        }

        private void TickBody()
        {
            // X0 한 워드(16비트) 읽기
            int s;
            if (!TryReadSensors(out s))
            {
                label1.Text = "PLC 통신 대기/재시도 중";
                return;
            }

            // 비트별 센서 상태
            bool bFwdDone = (s & 0x0004) != 0; // X02 B전진완료
            bool bBackDone = (s & 0x0008) != 0; // X03 B후진완료
            bool cFwdDone = (s & 0x0010) != 0; // X04 C전진완료
            bool cBackDone = (s & 0x0020) != 0; // X05 C후진완료
            bool liftAUp = (s & 0x0040) != 0; // X6 리프트A 상승완료
            bool liftADown = (s & 0x0080) != 0; // X7 리프트A 하강완료
            bool liftBUp = (s & 0x0100) != 0; // X8 리프트B 상승완료
            bool liftBDown = (s & 0x0200) != 0; // X9 리프트B 하강완료
            bool liftAOn = (s & 0x0400) != 0; // XA 리프트센서 A
            bool liftBOn = (s & 0x0800) != 0; // XB 리프트센서 B

            // 라벨에 현재 상태 표시 (자동/수동 둘 다)
            label2.Text = "리프트센서 A: " + (liftAOn ? "ON (감지)" : "OFF");
            label3.Text = "리프트센서 B: " + (liftBOn ? "ON (감지)" : "OFF");
            label4.Text = "B실린더: " + (bFwdDone ? "전진 완료" : bBackDone ? "후진 완료" : "동작 중");
            label5.Text = "C실린더: " + (cFwdDone ? "전진 완료" : cBackDone ? "후진 완료" : "동작 중");
            label6.Text = "리프트A: " + (liftAUp ? "상승 완료" : liftADown ? "하강 완료" : "동작 중");
            label7.Text = "리프트B: " + (liftBUp ? "상승 완료" : liftBDown ? "하강 완료" : "동작 중");

            if (!autoMode) return;

            // 자동운전 순서:
            // B실린더 전진 -> 리프트B 센서 감지 -> 리프트B 하강
            // -> C실린더 전진 -> 리프트A 센서 감지 -> 리프트A 상승 -> 반복
            switch (autoStep)
            {
                // B실린더 후진해서 원점 맞추기
                case AutoStep.HomeB:
                    if (bBackDone)
                    {
                        autoStep = AutoStep.HomeC;
                        break;
                    }
                    WriteY(Y_B_BACKWARD);
                    label1.Text = "자동: B실린더 후진 원점";
                    break;

                // C실린더 후진해서 원점 맞추기 (이미 물체 있으면 그 단계로 점프)
                case AutoStep.HomeC:
                    if (liftBOn && liftBDown) autoStep = AutoStep.ReadyLiftADown;
                    else if (liftBOn) autoStep = AutoStep.MoveLiftBDown;
                    else if (liftAOn && !liftAUp) autoStep = AutoStep.RetractCAfterDetect;
                    else if (cBackDone) autoStep = AutoStep.ReadyLiftBUp;
                    else
                    {
                        WriteY(Y_C_BACKWARD);
                        label1.Text = "자동: C실린더 후진 원점";
                    }
                    break;

                // 리프트B 올려서 받을 준비
                case AutoStep.ReadyLiftBUp:
                    if (liftBOn) autoStep = AutoStep.RetractBAfterDetect;
                    else if (liftBUp) autoStep = AutoStep.PushToLiftB;
                    else
                    {
                        WriteY(Y_LIFTB_UP);
                        label1.Text = "자동: 리프트B 상승(받기 준비)";
                    }
                    break;

                // B실린더 전진해서 센서B 켜질 때까지 밀기
                case AutoStep.PushToLiftB:
                    if (liftBOn) autoStep = AutoStep.RetractBAfterDetect;
                    else
                    {
                        WriteY(Y_B_FORWARD);
                        label1.Text = "자동: B실린더 전진(센서B 감지 대기)";
                    }
                    break;

                // 센서B 감지됐으면 B실린더 후진
                case AutoStep.RetractBAfterDetect:
                    if (bBackDone) autoStep = AutoStep.MoveLiftBDown;
                    else
                    {
                        WriteY(Y_B_BACKWARD);
                        label1.Text = "자동: B실린더 후진";
                    }
                    break;

                // 리프트B 내려서 물체 옮기기
                case AutoStep.MoveLiftBDown:
                    if (liftBDown) autoStep = AutoStep.ReadyLiftADown;
                    else
                    {
                        WriteY(Y_LIFTB_DOWN);
                        label1.Text = "자동: 리프트B 하강";
                    }
                    break;

                // 리프트A 내려서 받을 준비
                case AutoStep.ReadyLiftADown:
                    if (liftAOn) autoStep = AutoStep.RetractCAfterDetect;
                    else if (liftADown) autoStep = AutoStep.PushToLiftA;
                    else
                    {
                        WriteY(Y_LIFTA_DOWN);
                        label1.Text = "자동: 리프트A 하강(받기 준비)";
                    }
                    break;

                // C실린더 전진해서 센서A 켜질 때까지 밀기
                case AutoStep.PushToLiftA:
                    if (liftAOn) autoStep = AutoStep.RetractCAfterDetect;
                    else
                    {
                        WriteY(Y_C_FORWARD);
                        label1.Text = "자동: C실린더 전진(센서A 감지 대기)";
                    }
                    break;

                // 센서A 감지됐으면 C실린더 후진
                case AutoStep.RetractCAfterDetect:
                    if (cBackDone) autoStep = AutoStep.MoveLiftAUp;
                    else
                    {
                        WriteY(Y_C_BACKWARD);
                        label1.Text = "자동: C실린더 후진";
                    }
                    break;

                // 리프트A 올려서 물체 옮기고 처음으로 돌아감
                case AutoStep.MoveLiftAUp:
                    if (liftAUp) autoStep = AutoStep.ReadyLiftBUp;
                    else
                    {
                        WriteY(Y_LIFTA_UP);
                        label1.Text = "자동: 리프트A 상승";
                    }
                    break;
            }
        }

        // X0 읽기. 통신 에러나면 false 주고 다음 틱에 다시 시도
        private bool TryReadSensors(out int value)
        {
            value = 0;
            if (!plcConnected || closing) return false;

            try
            {
                short readValue;
                int result = control.ReadDeviceBlock2("X0", 1, out readValue);
                if (result != 0) return false;

                value = (int)(ushort)readValue;
                return true;
            }
            catch
            {
                return false;
            }
        }

        // Y0 쓰기. 직전 값이랑 같으면 그냥 넘어가서 통신 횟수 줄임
        private bool WriteY(short word, bool force = false)
        {
            if (!plcConnected) return false;
            if (closing && !force) return false;
            if (!force && word == lastWrittenOutput) return true;

            try
            {
                short writeValue = word;
                int result = control.WriteDeviceBlock2("Y0", 1, ref writeValue);
                if (result != 0) return false;

                output = word;
                lastWrittenOutput = word;
                return true;
            }
            catch
            {
                return false;
            }
        }

        // onBit는 켜고 offBit는 끔. 나머지 출력은 그대로 둠
        private void SetOutput(int onBit, int offBit)
        {
            // Y0 다시 안 읽고 output 값으로 해당 비트만 바꿈
            // (ushort로 안 바꾸면 short 부호 때문에 비트연산이 이상해짐)
            ushort cur = (ushort)output;
            cur = (ushort)((cur | onBit) & ~offBit);
            WriteY((short)cur);
        }

        private void button7_Click(object sender, EventArgs e)
        {
            RunManualCommand(Y_LIFTB_UP, Y_LIFTB_DOWN, label7, "리프트B: 상승 신호 ON");

        private void button8_Click(object sender, EventArgs e)
